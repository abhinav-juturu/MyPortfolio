Smart String Search Engine

**Core Technologies:** C++, Trie, Binary Search Tree (BST)

Project Goal & Summary

The primary goal was to develop a **prefix-based string search system** that mimics the functionality of modern search engines, providing real-time suggestions as the user types. This required an efficient data structure capable of fast traversal based on shared prefixes.

Data Structures Used

The core of the system relies on specialized data structures:

  • **Trie (Prefix Tree):** Used for **efficient insertion and retrieval** of words based on common prefixes.
  • **Binary Search Tree (BST):** Incorporated to manage and quickly traverse specific subsets of data.

Results & Optimizations

  • **Reduced Search Latency:** Optimized traversal methods drastically minimized the time taken to generate suggestions.
  • **Improved Accuracy:** The prefix-based nature ensures suggestions are highly relevant.
  • **Memory Optimization:** Efficient C++ implementation ensured minimal memory usage.

Outcome: Provided improved suggestion accuracy and reduced search latency by 40% (estimated).
